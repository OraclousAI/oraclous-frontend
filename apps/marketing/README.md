# @oraclous/marketing

The Oraclous marketing site (`oraclous.com`) ‚Äî an **Astro static site (SSG)** that renders the
content the marketing team authored under `oraclous-frontend/marketing/`. It is separate from the
authenticated console (`apps/console`, served at `app.oraclous.com`).

Why a separate static app: AI crawlers and search engines need crawlable HTML. The console is a
client-rendered SPA (correct for an auth-walled app, wrong for an indexable marketing surface). This
app pre-renders every page to static HTML with full meta + JSON-LD ‚Äî the AEO/GEO foundation.

See the full blueprint: [`../../marketing/build-spec/marketing-app-architecture.md`](../../marketing/build-spec/marketing-app-architecture.md).

## Develop

```bash
# Build the design system once (the app consumes its tokens):
pnpm --filter @oraclous/design-system build

# Dev server:
pnpm --filter @oraclous/marketing dev

# Production build + local preview:
pnpm --filter @oraclous/marketing build
pnpm --filter @oraclous/marketing preview
```

> Build-order gotcha (same as the console): `@oraclous/ui-utils` ‚Üí `@oraclous/design-system` ‚Üí
> `@oraclous/marketing`. `pnpm -r build` from the repo root orders this automatically.

## How it works

- **Content stays canonical in `marketing/`.** `src/content.config.ts` reads `marketing/pages/**`
  and `marketing/blog/**` in place via a glob loader (no fork of the content). Front-matter is
  validated by a Zod schema.
- **Routing follows the front-matter `url`.** `src/pages/[...slug].astro` resolves every non-home
  page by its `url`; `index.astro` is the home; `blog/` handles the blog index + posts.
- **SEO is by construction.** `BaseHead.astro` emits title/description/canonical/OG/Twitter from
  front-matter; `SchemaJsonLd.astro` emits JSON-LD from each page's `schema[]` array (BreadcrumbList
  on every page; Organization/WebSite/SoftwareApplication/FAQPage/Article/TechArticle/HowTo as
  declared). The FAQ JSON-LD is parsed from the body's `**Q:** / A:` blocks (single source).
- **Authoring trailers are stripped.** A remark plugin in `astro.config.mjs` removes the
  `**Internal links:**` / `**Notes for build:**` trailers and `(build note)` blockquotes before
  render, while keeping the `> **Citable answer**` blocks.
- **Styling is token-first.** Imports `@oraclous/design-system/tokens.css` + the self-hosted fonts;
  no Tailwind. Components are Astro + scoped CSS using the design-system CSS variables.
- **GEO assets** (`robots.txt`, `llms.txt`, `llms-full.txt`) ship from `public/`; `sitemap-index.xml`
  is generated by `@astrojs/sitemap`.
